# 将财务工具部署为可分享的网页链接（完整教程）

## 你需要准备的

- 一台能上网的电脑
- 一个邮箱（QQ邮箱、公司邮箱都行）
- 工具代码文件（ZIP包解压后的两个文件夹）
- 大约20分钟时间

---

## 什么是部署？为什么要部署？

你目前的情况是：在VS Code里运行 `streamlit run app.py`，工具只能在你自己的电脑上用。

**部署**就是把工具放到互联网上的一台服务器上，让它变成一个**链接**，任何人打开浏览器就能用。

部署后，你会得到类似这样的链接：
```
https://cscec-receivable.streamlit.app
```

把这个链接发给领导、同事，他们**不需要安装任何软件**，点链接就能直接用。

---

## 整体流程（先看一遍再动手）

1. 注册 GitHub 账号（代码托管平台，相当于存放代码的网盘）
2. 创建两个仓库（一个放回款诊断器代码，一个放模板生成器代码）
3. 上传代码文件（把电脑上的代码传到GitHub）
4. 注册 Streamlit Cloud（免费部署平台）
5. 部署第一个工具（回款诊断器）
6. 部署第二个工具（模板生成器）
7. 得到两个链接，分别发给需要的人

**两个工具需要分别部署**，得到两个独立的链接。

## 第1步：注册 GitHub 账号

### 1.1 打开 GitHub 网站
1. 打开电脑上的浏览器（Chrome、Edge、360都可以）
2. 在地址栏输入 `https://github.com/` 然后回车
3. 等待页面加载，你应该看到一个深色背景的页面，中间有一些代码片段在动

### 1.2 开始注册
1. 看页面正中间偏上的位置，有一行大字 "Let's build from here"
2. 在这行字的**右下方**，有一个**绿色的按钮**，上面写着 **"Sign up for GitHub"**
3. 点击这个绿色按钮

### 1.3 输入邮箱
1. 页面会跳转到一个新页面，标题是 "Create your account"
2. 第一个输入框是 "Enter your email"，填你的邮箱：
   - 可以用QQ邮箱（如 `123456@qq.com`）
   - 可以用公司邮箱
   - 建议用常用邮箱，因为后面要收验证码
3. 填好后，点击下面的 **"Continue"** 按钮（蓝色的）

### 1.4 设置密码
1. 第二个页面标题是 "Create a password"
2. 输入框里填密码，要求：
   - 至少8位
   - 包含英文字母
   - 包含阿拉伯数字
   - 示例：`Beijing2024`
3. 填好后，点 **"Continue"**（蓝色按钮）

### 1.5 设置用户名
1. 第三个页面标题是 "Enter a username"
2. 这个用户名**非常重要**，后面部署的时候会用到，要记下来！
3. 输入一个英文用户名，规则：
   - 只能用英文字母、数字、连字符 `-`
   - 不能以连字符开头或结尾
   - 示例：`zhangsan-cscec`、`liwei-finance`、`wangwu2024`
4. 如果用户名被占用了，页面会提示 "Username is already taken"，换一个再试
5. 成功后点 **"Continue"**

### 1.6 完成人机验证
1. 页面会显示一个验证题，比如：
   - "Pick the image with a bicycle"（选出有自行车的图片）
   - 或者 "Pick the image with a crosswalk"（选出有人行横道的图片）
2. 按照文字描述，**点击对应的图片**
3. 选对了会自动进入下一步，选错了会换一题

### 1.7 创建账号
1. 验证通过后，页面显示 "Create account"
2. 点击绿色的 **"Create account"** 按钮

### 1.8 邮箱验证（关键步骤）
1. 页面显示 "Please verify your email address"，告诉你验证码发到了邮箱
2. **打开一个新的浏览器标签页**，登录你的邮箱：
   - 如果是QQ邮箱：打开 `mail.qq.com`，扫码或密码登录
   - 如果是公司邮箱：打开公司邮箱网页
3. 在收件箱里找一封新邮件，**发件人是 noreply@github.com**，标题是 **"[GitHub] Please verify your email address"**
4. 打开这封邮件，正文里有一行："Your verification code is: **123456**"（6位数字）
5. 复制这6位数字
6. 回到 GitHub 注册页面，在 "Enter code" 输入框里粘贴验证码
7. 点击 **"Continue"**
8. 看到 "Welcome to GitHub" 和一个调查问卷页面，说明你注册成功了！

### 1.9 跳过问卷
1. 页面会问你一些问题（比如有多少人用、你是学生还是工作）
2. 随便选就行，或者直接点页面底部的 **"Skip personalization"**（跳过）
3. 进入 GitHub 首页

> **重要提醒**：
> - 你的用户名（如 `zhangsan-cscec`）要记下来！后面会用到。
> - 你的密码也要记住。
> - 建议把用户名写在记事本里。

## 第2步：创建 GitHub 仓库（回款诊断器）

仓库就是一个存放代码的文件夹。我们需要创建一个仓库来存放回款诊断器的代码。

### 2.1 打开创建页面
1. 确保你在 GitHub 首页（页面中间有 "Start writing code" 等字样）
2. 看页面的**最左上角**，有一个 **GitHub 的猫形 logo**
3. 在这个 logo 的**右边**，有一个 **+ 号按钮**（圆形的，里面有加号）
4. **鼠标放在这个 + 号上**（不要点），会弹出一个下拉菜单
5. 在下拉菜单里，点第一个选项 **"New repository"**（新建仓库）

### 2.2 填写仓库信息
页面跳转到 "Create a new repository"，从上到下依次填写：

**Repository name（仓库名字）**：
- 在输入框里填：`cscec-receivable`
- 这个名字的意思是：中建一局-回款诊断器
- **只能用英文、数字、连字符**

**Description（描述，可选）**：
- 可以不填
- 或者填："业主回款健康度诊断器"

**Public / Private（公开还是私有）**：
- 点选 **"Public"**（单选按钮）
- 必须是 Public 才能免费部署

**三个复选框（都不要选）**：
- "Add a README file" —— **不要勾选**
- "Add .gitignore" —— **不要勾选**
- "Choose a license" —— **不要勾选**

### 2.3 创建仓库
1. 页面最下方，有一个**绿色的大按钮** **"Create repository"**
2. 点击这个按钮
3. 页面会跳转，显示一个新的页面
4. 看到页面标题是 `你的用户名/cscec-receivable`，就创建成功了

### 2.4 记住这个页面
创建成功后，页面中间有一些蓝色的链接，包括：
- "uploading an existing file"
- "creating a new file"
- "..."

**不要关这个页面，下一步直接在这个页面上操作。**

## 第3步：上传代码文件到 GitHub

现在要把你电脑上的回款诊断器代码传到这个仓库里。

### 3.1 找到上传入口
在上一步的仓库页面（标题是 `你的用户名/cscec-receivable`）：
1. 看页面**中间位置**，有一行文字
2. 找到蓝色链接 **"uploading an existing file"**（在 "...or" 后面）
3. 点击这个蓝色链接
4. 页面跳转，显示一个大的虚线框，写着 "Drag files here to add them to your repository..."

### 3.2 上传第一组文件（代码文件）

**准备**：打开你的电脑上的文件管理器，找到解压后的 "业主回款健康度诊断器" 文件夹。

**上传 app.py**：
1. 在 GitHub 页面上，点击虚线框下面的 **"choose your files"** 蓝色文字
2. 弹出文件选择窗口
3. 找到 "业主回款健康度诊断器" 文件夹，双击打开
4. 选中 `app.py` 这个文件
5. 点文件选择窗口的 **"打开"** 按钮
6. 等待文件上传完成，在虚线框下面会显示文件名和大小

**上传 receivable_analyzer.py**：
1. 继续点击 **"choose your files"**
2. 选中 `receivable_analyzer.py`
3. 点 "打开"
4. 等待上传

**上传 requirements.txt**：
1. 继续点击 **"choose your files"**
2. 选中 `requirements.txt`
3. 点 "打开"

**上传 config.toml**（在隐藏文件夹里）：
1. 继续点击 **"choose your files"**
2. 进入 `.streamlit` 文件夹（如果看不到，说明这是隐藏文件夹）
3. 选中 `config.toml`
4. 点 "打开"

> 如果看不到 `.streamlit` 文件夹：
> - Windows：在文件选择窗口的地址栏输入文件夹路径
> - 或者直接把整个文件夹拖上去

### 3.3 上传第二组文件（图片资源）

需要上传 `assets` 文件夹里的所有图片。最简单的方法是**整个文件夹拖上去**：

1. 打开文件管理器，找到 "业主回款健康度诊断器" 文件夹
2. 把里面的 `assets` 文件夹（整个文件夹，不是里面的单个文件）
3. **用鼠标拖住，拖到浏览器的虚线框里，松手**
4. 等待上传完成，会看到：
   ```
   assets/logo.png
   assets/banner.png
   assets/construction_icon.png
   ```

如果拖不上去，就逐个上传：
1. 点击 **"choose your files"**
2. 进入 `assets` 文件夹
3. 选中所有图片（按住 Ctrl 键点多个文件）
4. 点 "打开"

### 3.4 确认文件列表

所有文件上传完成后，在虚线框下面应该显示以下文件：
```
app.py
receivable_analyzer.py
requirements.txt
config.toml
assets/logo.png
assets/banner.png
assets/construction_icon.png
```

如果有漏掉的，重复上面的步骤补传。

### 3.5 提交上传
1. 看页面**下方**，有一个区域写着 **"Commit changes"**
2. 第一个输入框里有默认文字，**不用改**
3. 直接点击下面的 **绿色按钮** **"Commit changes"**
4. 等待页面刷新
5. 刷新后，页面显示文件列表，能看到你上传的所有文件，说明成功了

## 第4步：注册 Streamlit Cloud

Streamlit Cloud 是一个免费的部署平台，它会自动把你的代码变成可访问的网页。

### 4.1 打开 Streamlit Cloud
1. 新建一个浏览器标签页（按 Ctrl+T）
2. 地址栏输入 `https://streamlit.io/cloud`
3. 回车，等待页面加载
4. 页面标题是 "Streamlit Cloud"，中间有一些介绍文字

### 4.2 开始注册
1. 页面中间偏上有一个**紫色按钮** **"Get started"**（也可能是 "Sign up"）
2. 点击这个紫色按钮

### 4.3 用 GitHub 登录
1. 页面显示几个登录方式
2. 找到 **"Continue with GitHub"** 按钮（紫色背景，白色文字）
3. 点击这个按钮

### 4.4 授权
1. 如果弹出一个页面问你是否授权 Streamlit 访问你的 GitHub 账号
2. 页面底部有一个**绿色按钮** **"Authorize streamlit"**
3. 点击授权
4. 页面跳转到 Streamlit Cloud 工作区

### 4.5 创建工作区（Workspace）
如果是第一次使用：
1. 页面让你创建一个 workspace
2. 在输入框里填一个名字，比如：`cscec-workspace`
3. 点击 **"Create workspace"**
4. 进入工作区首页

如果不是第一次：
1. 直接进入工作区首页
2. 看到 "Your apps" 等字样

## 第5步：部署回款诊断器

### 5.1 创建新应用
1. 在 Streamlit Cloud 工作区页面
2. 看页面**右上角**，有一个 **"New app"** 按钮（蓝色的）
3. 点击 **"New app"**
4. 弹出一个窗口，标题是 "Deploy an app"

### 5.2 填写部署配置
在弹出窗口里，从上到下依次选择：

**Repository（仓库）**：
1. 第一个下拉框，点击展开
2. 在下拉列表里找到你的仓库：`你的用户名/cscec-receivable`
3. 如果没有看到，等10秒再点，或者刷新页面重试
4. 点击选中这个仓库

**Branch（分支）**：
1. 第二个下拉框，点展开
2. 选 `main`（只有一个选项）

**Main file path（主文件路径）**：
1. 输入框里填：`app.py`
2. 这是告诉 Streamlit 运行哪个文件

**App URL (optional)**：
1. 输入框里填：`cscec-receivable`
2. 这样你的链接就是 `https://cscec-receivable.streamlit.app`
3. 也可以不填，系统会自动生成一个随机名字

### 5.3 开始部署
1. 确认上面的信息都填好了
2. 点击弹出窗口最下方的 **"Deploy!"** 按钮（紫色背景，白色文字）
3. 弹出窗口关闭，页面跳转到部署状态页面

### 5.4 等待部署完成
部署状态页面会显示几个步骤，依次执行：

| 状态 | 说明 |
|------|------|
| ⏳ Spinning up manager... | 启动管理器 |
| ⏳ Cloning repository... | 从GitHub下载代码 |
| ⏳ Building app... | 构建应用 |
| ⏳ Downloading dependencies... | 下载pandas、plotly等库 |
| ✅ Your app is in the oven! | 正在构建中，等2-3分钟 |

**你需要做的就是：等待。**

大概 **2-5分钟** 后，变成：
```
✅ Your app is ready!
```

这就是部署成功了！

### 5.5 获得链接
部署成功后：
1. 页面顶部显示你的应用链接，格式：
   ```
   https://cscec-receivable.streamlit.app
   ```
   （如果你没填App URL，会是随机的名字）
2. 点击这个链接，新标签页打开
3. 应该能看到回款诊断器的界面

### 5.6 测试
1. 在浏览器里打开你的链接
2. 上传示例数据 `FBL5N示例数据.xlsx` 测试
3. 确认功能正常

> **第一个工具部署完成！** 把链接保存下来，后面发给领导。

# 常见问题排查

---

## 问题1：部署时提示 "Repository not found"

原因：Streamlit Cloud 没有权限访问你的GitHub仓库  
解决：
1. 刷新 Streamlit Cloud 页面
2. 或退出重新登录
3. 确保你选的是自己的仓库

---

## 问题2：部署后页面显示 "Error"

原因：代码有错误，或者文件缺失  
解决：
1. 在 Streamlit Cloud 页面点 "Manage app" → "Logs"
2. 看报错信息
3. 常见原因：
   - 文件没传全 → 回GitHub补传
   - requirements.txt 依赖不对 → 检查内容
   - 代码有bug → 修改后重新Commit

---

## 问题3：链接在国内打不开

原因：Streamlit Cloud 的域名在国内部分地区被拦截  
解决：
1. 换手机流量试试
2. 或等几分钟再试
3. 或使用代理/VPN
4. 备选方案：部署到 Hugging Face（https://huggingface.co/spaces）

---

## 问题4：图片显示不出来

原因：assets 文件夹没上传，或者路径不对  
解决：
1. 检查 GitHub 仓库里是否有 `assets/` 文件夹
2. 检查文件夹里是否有图片文件
